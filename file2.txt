qué son las ramas: Branch
veamos esta metáfora, cada commit es como una fotograma de una película, donde estamos parados (último commit) es HEAD

es como una ruta que cada persona o equipo comienza a modificar sobre el proyecto, un commit y podemos ramificarlo
luego toda ramificación (branch) debe juntarse en una sola línea: Merge commit

git branch nos dirá en qué rama estamos / la principal es la rama master


git checkout -b [nombre de la nueva rama] / aquí podemos decir que crearemos una rama nueva, el b crea la nueva rama, para la siguiente solo debemos colocar checkout y el nombre de la rama
